## 📘 README – Kanban + GitHub (EverPia)

### 📝 Breu descripció
Aquest producte combina dues eines essencials per al treball professional dins la consultora **EverPia**:

- 📋 **Gestió visual de tasques** amb metodologia Kanban mitjançant Microsoft Planner  
- 💻 **Repositori personal a GitHub**, que actua com a espai individual de treball i documentació  

Amb aquesta combinació, cada grup i cada alumne disposa d’un sistema d’organització **clar, transparent i professional**, similar al d’un equip real del sector IT.

---

## 🎯 Objectius del producte
- Garantir que cada grup disposa d’una eina de **coordinació col·laborativa (Planner)**
- Assegurar que cada alumne té un **espai individual de treball (GitHub)**
- Reforçar la metodologia professional d’**EverPia**:
  - Transparència  
  - Organització  
  - Traçabilitat del treball  

---

## 📋 Part 1 – Tauler Kanban (Microsoft Planner)

Cada grup ha de crear un tauler Kanban amb les següents característiques:

### 🧩 Columnes obligatòries
- **Backlog** → totes les tasques del projecte  
- **En curs** → tasques en desenvolupament  
- **En revisió** → tasques pendents de validació  
- **Fet** → tasques completades  

### ⚙️ Funcionament
- El **Backlog** ha d’estar complet des del principi amb totes les tasques (T01, T02… T10)
- Les tasques es mouen entre columnes segons el progrés
- El tauler ha d’estar sempre **actualitzat**

### ✅ Lliurable
- Enllaç públic o compartit del **tauler Kanban**

---

## 💻 Part 2 – Compte i repositori GitHub (individual)

Cada alumne ha de:

### 🔧 Configuració
- Crear un compte de **GitHub**
- Associar-lo amb el correu de l’escola

### 📁 Repositori personal
- Crear un repositori de presentació
- Escriure un fitxer `README.md` amb:
  - Presentació personal
  - Informació rellevant

### ✅ Lliurable
- Enllaç al **repositori personal de GitHub**
